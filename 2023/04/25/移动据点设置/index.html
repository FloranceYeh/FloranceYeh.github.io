移动据点设置指南

已经成为历史

因为我已经写了自动部署

毕竟是历史，留个纪念

由于Florance的一个据点被端掉了，于是我不得不写这个博文，以方便我配置2号据点（2号据点每日都会被扫荡，于是每次进入都得重新配置，这也是1号据点被端后我很烦的原因）

同时，这个据点设置指南全是我自己总结的，所以比较稳定，至少不会被 port xxx reset connection

因此，此指南不仅可以用于固定据点设置，也可用于移动据点的设置

恼

  1. 安装 Git

  2. 安装 node.js

  3. 链接 Github 账号

    1. git config --global user.name "FloranceYeh"
    2. git config --global user.email "Florance_Sunrise@outlook.com"

  4. 生成 ssh key

    1. ssh-keygen -t rsa -C "Florance_Sunrise@outlook.com"

  5. Github 链接 ssh key

    1. cat ~/.ssh/id_rsa.pub
    2. https://github.com/settings/keys to

  6. 检查链接

    1. ssh -T git@github.com
    2. 若22端口失败，配置443端口
      1. vim ~/.ssh/config
      2. 编写 config 里的内容
      Host github.com
    User Florance_Sunrise@outlook.com
    Hostname ssh.github.com
    PreferredAuthentications publickey
    IdentityFile ~/.ssh/id_rsa
    Port 443

  7. 再次检查链接

    1. ssh -T git@github.com

  8. 配置HEXO

    1. cd D:\blog
    2. npm install hexo-cli -g
    3. npm install

  9. 如果又被 reset 了，删除 known_hosts 重试

    1. C:\Users\admin\.ssh\known_hosts

最后的最后，每周战略性转移时记得打包行李！
